# 前端大改革方案

## 当前现状

本项目当前是 `PHP 管理后台 + Vue 静态前端产物 + Node WebSocket 控制通道 + MySQL 数据库 + APK 构建工具链`。现有 `assets/` 只有 Vite 打包后的 JS/CSS，没有 Vue 源码、`package.json`、`src/`、`vite.config.ts`，因此不建议继续直接修改打包产物。

结论：这不是“小修前端”，而是一次重建前端与后端的迁移。

## 目标技术栈

- 前端用户端：Vue 3 + TypeScript + Vite
- 前端管理端：Vue 3 + TypeScript + Vben Admin + Ant Design Vue
- 后端：Java 21，建议 Spring Boot 3
- 实时通道：Java WebSocket 或保留 Node WebSocket 作为过渡层
- 数据库：MySQL/MariaDB，SQL schema 与 migrations 独立管理
- 包管理：前端统一使用 `pnpm`

## 推荐目录结构

```text
MM-v2/
  apps/
    web/                  # 用户端：登录、展示页等
    admin/                # 管理端：设备列表、控制台、构建管理
  backend/
    src/main/java/        # Java 21 后端
    src/main/resources/
    pom.xml
  database/
    schema/
    migrations/
    seed/
  docs/
    api.md
    websocket.md
    deploy.md
```

如果必须控制成三套目录，可以把 `apps/web` 和 `apps/admin` 合并到 `frontend/` 下，但管理后台仍应独立路由和独立权限。

## 页面拆分

当前至少有三类界面：

- 登录页：对应当前 `/` 登录入口
- 登录后展示页：设备列表、账号信息、构建信息等
- 控制界面：对应当前 `/info?id=设备ID`

新系统建议路由：

```text
/login
/dashboard
/devices
/devices/:id/control
/apps/build
/settings/security
```

## 后端迁移原则

不要一次性删除 PHP。建议先用 Java 复刻接口，再逐步切流。

第一阶段保留：

- PHP API：`api/auth.php`、`api/devices.php`、`api/detail.php`
- Node WebSocket：`api/ws/websocket-server.js`
- MySQL 表结构：先从 `1.sql` 整理出 schema

第二阶段替换：

- `api/auth.php` -> `POST /api/auth/login`
- `api/devices.php` -> `GET /api/devices`
- `api/detail.php` -> `GET /api/devices/{id}`
- `api/toggle.php` -> `PATCH /api/devices/{id}/status`
- `api/update_remark.php` -> `PATCH /api/devices/{id}/remark`

第三阶段再处理 APK 构建链：

- PHP `private/build_api.php`
- PHP `private/build_runner.php`
- `.NET` 构建程序 `EaodStarter` / `EaodWorker`
- Python 工具 `private/tools/*.py`

## 安全清理优先级

重构前先处理高风险逻辑，否则新前端会继续接入旧后门：

1. 删除 `app.config.js` 中 WebSocket `change -> reassign` 的拦截逻辑。
2. 禁用后端 `reassign` WebSocket 分支，除非重新设计明确权限。
3. 删除 `WS_SUPER_EMAIL_HASH` 超级邮箱入口。
4. 强制配置 `WS_ADMIN_TOKEN`，不能空 token 依赖内网 IP 放行。
5. 清理旧服务器域名：`p4q2.cn`、`bb.6upw.com`、`yk.6668888.top`、`888.sfsa.top`。

## 数据库管理

把 `1.sql` 拆成：

```text
database/schema/001_base_schema.sql
database/migrations/002_add_indexes.sql
database/seed/local_admin.sql
```

不要把生产账号、旧域名路径、历史构建记录放进 seed。开发环境只保留本地 admin 和测试数据。

## 开发启动建议

```bash
pnpm create vite apps/web --template vue-ts
pnpm create vite apps/admin --template vue-ts
cd backend && ./mvnw spring-boot:run
```

Vben Admin 建议单独初始化到 `apps/admin`，不要混进用户端。

## 执行顺序

1. 先安全清理旧项目后门和旧域名。
2. 新建 `MM-v2` 目录，不在旧打包产物上继续开发。
3. 抽数据库 schema 和 API 文档。
4. 搭建 Java 21 后端认证、设备列表、设备详情接口。
5. 搭建 `apps/admin` 管理后台并接入 Java API。
6. 迁移 WebSocket 控制协议。
7. 最后迁移 APK 构建链。

## 代码规范

所有一方维护的源代码文件不超过 500 行。超过后必须按职责拆分，例如：

- Vue 页面拆成 `views/`、`components/`、`composables/`、`api/`
- Java 后端拆成 `controller`、`service`、`repository`、`dto`
- WebSocket 协议拆成连接管理、权限校验、消息分发、设备状态模块
